REST Endpoints and Useful Links

Admin Page
Link: /admin
Username: admin
Password: admin
Endpoint 1:
Name: Airlines List
Route: /api/airlines_list
Methods: GET
Details: This endpoint proves the user with a list of all airlines in the database, providing the primary key, and name of the airline.
Example Link: /api/airlines_list
Endpoint 2:
Name: Airline Detail
Route: /api/airline_info/[AIRLINE PRIMARY KEY]
Methods: GET, DELETE
Details: Using the primary key of the airline, this page displays all the information fields available in the model for that specific airline. The ability to delete the airline entry is also available.
Example Link: /api/airline_info/2
Endpoint 3:
Name: Airports List
Route: /api/airport_list
Methods: GET
Details: This endpoint proves the user with a list of all airports in the database, providing the primary key, and name and location of the airport.
Example Link: /api/airport_list
Endpoint 4:
Name: Airport Detail
Route: /api/airport_info/[AIRPORT PRIMARY KEY]
Methods: GET, DELETE
Details: Using the primary key of the airport, this page displays all the information fields available in the model for that airport. The ability to delete the airport is also available.
Example Link: /api/airline_info/3
Endpoint 5:
Name: Equipment List
Route: /api/equipment_list
Methods: GET
Details: This endpoint proves the user with a list of all airplanes in the database, providing the primary key, and name of the plane.
Example Link: /api/equipment_list
Endpoint 6:
Name: Equipment Detail
Route: /api/equipment_info/[AIRPLANE PRIMARY KEY]
Methods: GET, POST, PUT, DELETE
Details: Using the primary key of the airplane, this page displays all the information fields available in the model for that model of airplane. The ability to edit, delete, and add new entries is also available.
Example Link: /api/equipment_info/4
